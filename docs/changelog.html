Changelog

Version 1.14.6
April 2026
  • Android RTP-MIDI Network Session — connect to Mac, iPad, or other RTP-MIDI hosts over Wi-Fi without a USB cable. Toggle it under Settings → MIDI → RTP-MIDI Network Session.
  • RTP-MIDI device list now shows each peer's IP and endpoint, and lets you disconnect peers individually. Tapping Disconnect now reaches the remote within a few seconds (previously the remote could take up to a minute to time out).
  • Fixed an occasional startup crash on Android 12+ when the main thread was busy and the foreground audio service couldn't call startForeground in time.
  • Hardened the startup path against an unused AdMob initialization provider that could crash the process before the Flutter engine was ready.
  • Fixed a native "Reply already submitted" crash that some Android devices hit when reconnecting BLE MIDI after a disconnect.
  • Fixed a notification crash on Android 8.0 and earlier — the notification builder was incorrectly calling post-Oreo channel APIs on devices that don't support them.
Version 1.14.5
April 2026
  • Major Bluetooth MIDI compatibility improvements, especially on OnePlus, OxygenOS and similar Android builds. Several models where BLE MIDI would connect but silently fail to deliver MIDI bytes now work reliably: the plugin now re-wires the output port against the live GATT session after Android's mid-connect rebond churn.
  • Paired BLE MIDI devices are now visible in the device picker as soon as the app launches — no need to run a Bluetooth scan first just to reconnect the keyboard you already paired.
  • After three failed Bluetooth connection attempts in a session, SynthBridge now surfaces a compatibility dialog with the developer's email so you can report the specific device and we can investigate.
  • Fixed the "Bluetooth is off" detection: the prompt now appears instantly when you tap Scan with Bluetooth disabled, and cached Bluetooth devices are no longer shown while the adapter is off.
  • Fixed a native-side crash in the Android MIDI plugin's Bluetooth-central startup path (IllegalStateException: Reply already submitted) that could surface on first scan.
  • Under the hood: finer-grained analytics on the BLE connection funnel so we can attribute future failures to specific device models rather than guessing.
Version 1.14.3
April 2026
  • Redesigned the "Open MIDI File" dialog as clean option cards. The play / convert-to-audio choices now each have their own icon, title and one-line description instead of three buttons competing for space in the action row.
  • Added a Close button on the main action bar whenever a MIDI file is loaded — lets you unload the current file without having to stop playback first.
  • The audio export dialog now survives navigating to the SoundFont manager: switch source, come back, and continue with your previous format selection. The default format has been switched from WAV to AAC to match what most users actually want.
  • Added confirmation prompts before stopping a recording or opening a new MIDI file, so accidentally tapping either action no longer loses in-progress work.
  • Fixed the new-user tutorial: the "Global FX & Metronome" step's highlight ring now extends to cover the metronome button in the non-compact portrait layout, not just the four FX buttons.
  • Fixed a rare exception when the file picker was invoked twice in quick succession.
  • Fixed an overflow in the price/paywall card when the plan title was too long.
Version 1.14.2
April 2026
  • Audio Lab now exposes two independent buffer sliders: one for live performance and one for MIDI file playback. Keep performance tight while letting playback run glitch-free on slower devices.
  • The Android audio callback is now pinned to the high-performance (big) cores via the OS performance-hint API, reducing latency jitter during real-time playing.
  • The MIDI log page now scrolls as a single horizontal surface when rows are wider than the screen — all rows pan together instead of each line sliding on its own.
  • Polished the device connection dialog wording for clarity.
Version 1.14.1
April 2026
  • Fixed a bug where BLE MIDI keyboards already paired with the system could not be connected. The device list now correctly handles keyboards discovered by the app's own scan as well as ones Android had already enumerated.
  • Fixed a leak where every BLE scan added another device-list listener that was never cleaned up, causing the device list to refresh at an ever-growing rate after extended use.
Version 1.14.0
April 2026
  • Added 10 bundled classical MIDI demo songs so the app has something to play the first time you open the library — no setup required.
  • Added four portrait stage layouts (drum pad + wheels + keyboard, drum pad + dual keyboard, dual drum pad, wheels + dual keyboard). Switch any time in Settings → Keyboard layout.
  • Added keyboard label modes for the on-screen piano: show only the C keys, every note name, or numbered notation (1–7) with dots indicating the octave. Change in Settings → Keyboard labels.
  • Upgrading users now see a short tour introducing the new layout and label-mode features on first launch.
  • The in-app tutorial now has dedicated steps for the on-screen keyboard and the overall stage layout.
  • Fixed a Google Play subscription crash that could occur on some OnePlus devices (international edition).
  • Fixed a crash when plugging in MIDI devices that expose no input ports.
  • Fixed an occasional crash when swiping between tabs on the sound settings page.
